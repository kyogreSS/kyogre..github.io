差点弄混

Link的属性是to，Router和Route的属性是path。
path可以用通配符模糊匹配。根据阮一峰的博客，具体的规则是：

  • :paramName：paramName匹配URL的一个部分。直到遇到下一个/、?、#为止。这个路径参数可以通过this.props.params.paramName取出
  • ()：()表示URL的这个部分是可选的
  • *：*匹配任何字符，知道模式里面的下一个字符为止。匹配方式是非贪婪模式
  • **：**匹配任意字符，直到下一个/、？、#为止。匹配方式是贪婪模式

本以为Atom编辑器就这么告一段落了，结果还是有很多坑。我遇到了几个小问题，这里记录下。

自动换行问题

这个自动换行，是说我一行代码超过了当前页面，自动到下一行编辑，而不是出现滚动条，把不管多长的东西放在一行。
这个要在setting里的editor选项里，勾选soft wrap选项，这样：

这个大概的意思是：在超过屏幕长度的地方换行。如果soft wrap at preferred line length这个选项勾选了，就在超过这个选项设置的长度的地方换行。

markdown保存的时候后面空格自动去掉

markdown语法里，两个空格+回车是开始新的一行。然而有时候如果不设置，一保存那俩空格就没了。
这个要在packages里设置。打开setting里的packages选项，在core packages里面找whitespace这个包：

点setting，找到remove trailing whitespace和ensure single trailing newline这两个选项，把它们去掉。

明明有keep markdown line break whitespace这个选项，在markdwon文件里居然不生效，这是神奇。
这两句，remove trailing whitespace的意思是去掉行后面多余的空格。如果要给单一语言设置，用它给的方式。
ensure single trailing newline的意思是如果文档流在保存的时候没有新一行的标志（回车吧应该），就自动加上，如果多于一个，就只保留一个。
这两样设置好，应该就没问题了。弄懂这几个，如果其他的文件再出什么问题，再根据需要改就好了。

Atom使用心得

Posted on 2017-06-24

Atom的使用心得

作为一名新晋前端，一开始接触的编辑器是webstorm。虽然在搜索破解方法的时候觉得挺麻烦，但在使用中的体验真心不错。很多很多快捷键、自动提示等都安装好了，安安心心编代码即可。用了一段时间，又被安利了一波Atom，说什么插件强大，功能多多，符合程序员开源的特性，是轻量级的编辑器，什么文件都能写，速度又快云云。收下安利，下载之。用起来才发现，真JB麻烦，碰到了不少坑。一点点的踩过坑后，倒也觉得还好。这里记下Atom的一些使用心得，不全面，仅供参考。

Atom的问题之一：插件

这是Atom最主要的问题了。刚下下来的Atom是个纯净版的编辑器，可以说是想要的功能都没有，跟个记事本一样，完全不适合新手好么。一开始就推荐你用Atom并且不教你怎么安装的人，以后漂流瓶联系吧。
虽然atom提供了方便快捷的插件下载方式，在setting里有install选项，可以找到自己想要的包。
然而不用一下其他自动化的编辑器，鬼知道有什么包好用呀。
也好解决，去百度谷歌一下，有很多推荐插件，开开心心用官方渠道下载，岂不美哉。
结果是特喵的下不下来，几率性被墙！于是乎，即使是知道哪个包适合自己用，也很难下载。
主要坑在于：1、用什么插件。2、怎么下载。

用什么插件

这个插件分为几种：

  1. 样式类，代码美化、界面美化的。
  2. 简化操作类，代码补全、提示的。
  3. 工具类，git、terminal等。
  4. 其他乱七八糟的，比如程序员鼓励插件。

第一种，样式类的：

  • atom-beautify 顾名思义，代码美化，推荐的最多的一款。
  • linter 格式化？排查错误的。
  • linter-csslint 排查CSS错误的。
  • linter-eslint 排查JS错误的。
  • linter-ui-default linter的依赖包吧，下载linter就会提示安装的。
  • 还有题外话，主题，在Themes里，我用的ONE DARK，黑色。

第二种，简化操作的：

  • autoclose-html 自动闭合html标签。
  • emmet 自动生成前端的代码。比如html再按tab，就能生成一些必要的东西。

第三种，工具类的：

  • git-plus 在行里能写git命令，虽然安装了，很少用。
  • platformio-ide-terminal 快捷打开终端/CMD。
  • open-in-browser 在浏览器里打开你的网页！

第四种，其他的：

  • color-picker 取色器，前端用得到。
  • file-icons 文件小图标，让文件列表有更高的标识度。
  • minimap 小型代码预览图，用来跳转代码的。
  • markdown-preview-plus markdown预览。
  • activate-power-mode 代码跳跃！写汉字的时候跳动的有些厉害，晃眼。

安装的插件在菜单栏Packages里面，需要手动用的可以在里面找，或者右键菜单里找。要是不想要了，在Settings里的Packages里可以卸载、设置、失效。
